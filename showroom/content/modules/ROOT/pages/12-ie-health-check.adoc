The factory sensors are already running and publishing data. In this step you will watch the live data flow from sensor pods through MQTT.

== See sensor data in real-time

Each sensor pod publishes vibration (and optionally temperature) readings every 5 seconds. Tail the logs:

[source,bash]
----
oc logs -f deploy/machine-sensor-1 -n industrial-edge-stormshift-machine-sensor --tail=3
----

Expected output — JSON payloads with vibration readings:

[source,json]
----
{"sensor":"vibration","value":14.2,"timestamp":"2026-04-17T17:30:00Z","device":"pump-1","machine":"floor-1-line-1-extruder-1"}
----

Press `Ctrl+C` to stop. Try the second sensor:

[source,bash]
----
oc logs -f deploy/machine-sensor-2 -n industrial-edge-stormshift-machine-sensor --tail=3
----

== Check all sensor pods

[source,bash]
----
oc get pods -n industrial-edge-stormshift-machine-sensor --no-headers | grep machine-sensor
oc get pods -n industrial-edge-tst-all --no-headers | grep machine-sensor
----

All should show `Running 1/1`.

== Inspect the sensor configuration

Each sensor is configured via a `ConfigMap`. Check what data types are enabled:

[source,bash]
----
oc get configmap machine-sensor-1 -n industrial-edge-stormshift-machine-sensor -o yaml | grep SENSOR_
----

Note these key fields:

* `SENSOR_VIBRATION_ENABLED: "true"` — vibration is active
* `SENSOR_TEMPERATURE_ENABLED: "false"` — temperature is off (you will enable it in xref:15-ie-mlops.adoc[step 13])

== Verify MQTT is receiving data

The sensors publish to the AMQ Broker via MQTT. Check the broker is healthy:

[source,bash]
----
oc get activemqartemis -n industrial-edge-stormshift-messaging
oc get pods -l ActiveMQArtemis -n industrial-edge-stormshift-messaging
----

Check the messaging app (Node.js MQTT consumer) is connected:

[source,bash]
----
oc logs deploy/messaging -n industrial-edge-stormshift-messaging --tail=5
----
