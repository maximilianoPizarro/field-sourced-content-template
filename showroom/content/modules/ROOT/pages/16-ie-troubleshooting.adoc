= Step 14: Explore MinIO Data Lake

MinIO provides S3-compatible storage for the Industrial Edge data lake. Sensor data flows from Kafka into S3 buckets for long-term storage and ML model training.

== Open the MinIO Console

link:https://minio-console-industrial-edge-ml-workspace.{cluster_domain}[MinIO Console^]

Login:

* *Username:* `minioadmin`
* *Password:* `minioadmin`

== Browse the buckets

After logging in, you should see these buckets:

[cols="1,2", options="header"]
|===
| Bucket | Purpose

| `pipeline-bucket`
| ML training pipeline artifacts and intermediate data

| `user-bucket`
| Data scientist workspace for notebooks and experiments

| `anomaly-models`
| Trained anomaly detection model artifacts
|===

Click into each bucket to explore its contents. If the Camel K Kafka→S3 integration is running, you will see sensor data files accumulating.

== Check MinIO from the CLI

List buckets using the MinIO pod:

[source,bash]
----
oc exec deploy/minio -n industrial-edge-ml-workspace -- \
  mc alias set local http://localhost:9000 minioadmin minioadmin 2>/dev/null

oc exec deploy/minio -n industrial-edge-ml-workspace -- \
  mc ls local/
----

List objects in the anomaly-models bucket:

[source,bash]
----
oc exec deploy/minio -n industrial-edge-ml-workspace -- \
  mc ls local/anomaly-models/ 2>/dev/null || echo "Bucket empty or not yet populated"
----

== Check MinIO health

[source,bash]
----
oc get pods -l app=minio -n industrial-edge-ml-workspace
oc get svc minio -n industrial-edge-ml-workspace
oc get route minio-console -n industrial-edge-ml-workspace
----

== How data gets here

The data flow into MinIO:

. *Sensors* → publish MQTT → *AMQ Broker*
. *Camel K* bridges MQTT → *Kafka*
. *Camel K* integration reads from Kafka topics and writes to *MinIO S3* buckets
. *OpenShift AI* notebooks read training data from S3 and write trained models back

Check the Camel K integration status:

[source,bash]
----
oc get integration -n industrial-edge-tst-all --no-headers 2>/dev/null
oc get integration -n industrial-edge-data-lake --no-headers 2>/dev/null
----

If `kafka-to-s3-integration` shows `Running`, data is actively flowing to S3.

== ArgoCD view

See all Industrial Edge applications and their health:

[source,bash]
----
oc get applications -n openshift-gitops --no-headers | grep industrial-edge
----

All should show `Synced`. Components in `Progressing` are still initializing (Camel K builds take a few minutes on first deploy).
