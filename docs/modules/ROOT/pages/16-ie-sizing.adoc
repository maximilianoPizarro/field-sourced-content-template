Sizing estimates for scaling the Industrial Edge platform from a demo/PoC to a production deployment supporting *20,000 IoT devices*.

== Throughput Baseline (20K Devices)

[cols="1,1", options="header"]
|===
| Parameter | Value

| Devices | 20,000 IoT sensors (vibration + temperature)
| Publish interval | 5 seconds per device
| Message rate | 8,000 msg/sec
| Data rate (raw) | 1.6 MB/sec
| Daily volume | ~138 GB/day (~4.1 TB/month)
| With RF=3 | ~4.8 MB/sec write, ~414 GB/day on cluster
|===

== Key Architecture Decisions

[cols="1,2", options="header"]
|===
| Decision | Rationale

| KRaft mode (no ZooKeeper) | Reduces pod count, simplifies operations, removes ZK as SPOF
| 5 combined broker+controller nodes | Sufficient for 8K msg/sec with RF=3 and 24 partitions
| 8-node MinIO with erasure coding | EC:2 tolerates 2 failed nodes, 1.5Ti per node for data lake
| Hub + Edge topology | Network segmentation, edge isolation, independent scaling
| MirrorMaker2 with 8 tasks/source | Handles 13K msg/sec cross-cluster replication
| TLS + SCRAM-SHA-512 for Mirror | Secure external access with read-only ACLs
|===

== KRaft Configuration Example

Kafka clusters at 20K scale use KRaft mode with `KafkaNodePool` (combined broker+controller roles):

[source,yaml]
----
apiVersion: kafka.strimzi.io/v1beta2
kind: Kafka
metadata:
  name: factory-cluster
  annotations:
    strimzi.io/kraft: enabled
    strimzi.io/node-pools: enabled
spec:
  kafka:
    version: 3.7.0
    listeners:
      - name: plain
        port: 9092
        type: internal
        tls: false
      - name: tls
        port: 9093
        type: internal
        tls: true
    config:
      default.replication.factor: 3
      min.insync.replicas: 2
      num.partitions: 24
  entityOperator:
    topicOperator: {}
    userOperator: {}
---
apiVersion: kafka.strimzi.io/v1beta2
kind: KafkaNodePool
metadata:
  name: combined
  labels:
    strimzi.io/cluster: factory-cluster
spec:
  replicas: 5
  roles:
    - controller
    - broker
  storage:
    type: jbod
    volumes:
      - id: 0
        type: persistent-claim
        size: 500Gi
        class: gp3-csi
        deleteClaim: false
  resources:
    requests:
      cpu: "4"
      memory: 8Gi
    limits:
      cpu: "8"
      memory: 16Gi
  jvmOptions:
    -Xms: 6144m
    -Xmx: 6144m
----
