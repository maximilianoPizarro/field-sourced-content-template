== Native Metrics

Kafka brokers expose JMX metrics via the Prometheus JMX Exporter on port 9404:

[source,yaml]
----
apiVersion: kafka.strimzi.io/v1beta2
kind: Kafka
metadata:
  name: cdc-cluster
spec:
  kafka:
    metricsConfig:
      type: jmxPrometheusExporter
      valueFrom:
        configMapKeyRef:
          name: kafka-metrics-config
          key: kafka-metrics-config.yml
  kafkaExporter:
    topicRegex: ".*"
    groupRegex: ".*"
----

The Kafka Exporter pod exposes consumer group lag, topic offsets, and broker info on port 9404.

== Alerts

PrometheusRule definitions fire alerts when issues are detected:

[source,yaml]
----
apiVersion: monitoring.rhobs/v1
kind: PrometheusRule
metadata:
  name: kafka-cdc-alerts
  labels:
    openshift.io/user-monitoring: "true"
spec:
  groups:
    - name: kafka-cdc
      rules:
        - alert: KafkaConsumerLagHigh
          expr: kafka_consumergroup_lag > 1000
          for: 5m
          labels:
            severity: warning
          annotations:
            summary: "Consumer lag high"
        - alert: DebeziumDisconnected
          expr: debezium_postgres_Connected == 0
          for: 2m
          labels:
            severity: critical
        - alert: KafkaConnectTaskFailed
          expr: kafka_connect_worker_connector_failed_task_count > 0
          for: 1m
          labels:
            severity: critical
----

== Upgrades

Strimzi performs rolling updates automatically when you update the `Kafka` CR version field. Brokers are restarted one at a time, maintaining availability.

[source,yaml]
----
spec:
  kafka:
    version: 3.8.0  # bump from 3.7.0
----

The operator handles:

* Rolling restart of brokers (one at a time)
* Inter-broker protocol negotiation
* Zero-downtime for producers and consumers
