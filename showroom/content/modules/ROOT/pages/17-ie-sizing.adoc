== Key architecture decisions (20K)

[cols="1,2", options="header"]
|===
| Decision | Rationale

| KRaft mode (no ZooKeeper)
| Reduces pod count, simplifies operations, removes ZK as SPOF

| 5 combined broker+controller nodes
| Sufficient for 8K msg/sec with RF=3 and 24 partitions

| 8-node MinIO with erasure coding
| EC:2 tolerates 2 failed nodes, 1.5Ti per node for data lake

| Hub + Edge topology
| Network segmentation, edge isolation, independent scaling

| MirrorMaker2 with 8 tasks/source
| Handles 13K msg/sec cross-cluster replication

| KServe InferenceService for LLMs
| Integrates with RHOAI Playground; replaces standalone Deployments

| Operator memory overrides (CSV patch)
| COO, Kuadrant, Kiali require 1-2 Gi vs OLM defaults of 128-512 Mi
|===

== What you learned

* The demo cluster runs **129 workload pods** using **29 vCPU / 65 Gi** (requests) on 3 workers with 64 vCPU / 128 Gi each
* Several operators (COO, Kuadrant, Kiali) require **memory limit overrides** via CSV patches to prevent OOMKilled loops
* Mailpit needs at least **512 Mi** when processing continuous CV notification emails
* Platform resource requirements scale from 129 pods / 29 vCPU (demo) to 118+ pods / 157 vCPU (20K devices)
* Minimum demo cluster: **3 workers x 64 vCPU x 128 Gi** (192 vCPU / 384 Gi total)
* Production deployments require 2-4 OpenShift clusters depending on HA and DR requirements
