=== OOM Fixes Applied (Lessons Learned)

Several operators and workloads were OOMKilled with their default memory limits. The following overrides are required for a stable deployment:

[cols="2,1,1,2", options="header"]
|===
| Component | Original | Fixed To | Reason

| Mailpit | 128 Mi | 512 Mi | OOMKilled after 24 restarts processing CV notification emails
| Perses Operator (Cluster Observability Operator) | 512 Mi | 1 Gi | OOMKilled during operator startup
| Kuadrant Operator (Red Hat Connectivity Link) | 300 Mi | 2 Gi | OOMKilled during reconciliation loop
| Kiali | default | 2 Gi | OOMKilled when processing service mesh topology
| Granite LLM (llama.cpp via KServe) | N/A | 4 Gi req / 8 Gi lim | Migrated from Deployment to KServe InferenceService for RHOAI Playground
|===

IMPORTANT: These memory overrides must be applied via CSV patches (`oc patch csv`) or by updating the operator subscriptions. The overrides are **not** persisted through operator upgrades — reapply after each upgrade.

== Key Architecture Decisions

[cols="1,2", options="header"]
|===
| Decision | Rationale

| KRaft mode (no ZooKeeper) | Reduces pod count, simplifies operations, removes ZK as SPOF
| 5 combined broker+controller nodes | Sufficient for 8K msg/sec with RF=3 and 24 partitions
| 8-node MinIO with erasure coding | EC:2 tolerates 2 failed nodes, 1.5Ti per node for data lake
| Hub + Edge topology | Network segmentation, edge isolation, independent scaling
| MirrorMaker2 with 8 tasks/source | Handles 13K msg/sec cross-cluster replication
| TLS + SCRAM-SHA-512 for Mirror | Secure external access with read-only ACLs
| KServe InferenceService for LLMs | Integrates with RHOAI Playground; replaces standalone Deployments
| Operator memory overrides (CSV patch) | COO, Kuadrant, Kiali require 1-2 Gi vs OLM defaults
|===

== KRaft Configuration Example

Kafka clusters at 20K scale use KRaft mode with `KafkaNodePool` (combined broker+controller roles):

[source,yaml]
----
apiVersion: kafka.strimzi.io/v1beta2
kind: Kafka
metadata:
  name: factory-cluster
  annotations:
    strimzi.io/kraft: enabled
    strimzi.io/node-pools: enabled
spec:
  kafka:
    version: 4.0.0
    listeners:
      - name: plain
        port: 9092
        type: internal
        tls: false
      - name: tls
        port: 9093
        type: internal
        tls: true
    config:
      default.replication.factor: 3
      min.insync.replicas: 2
      num.partitions: 24
  entityOperator:
    topicOperator: {}
    userOperator: {}
---
apiVersion: kafka.strimzi.io/v1beta2
kind: KafkaNodePool
metadata:
  name: combined
  labels:
    strimzi.io/cluster: factory-cluster
spec:
  replicas: 5
  roles:
    - controller
    - broker
  storage:
    type: jbod
    volumes:
      - id: 0
        type: persistent-claim
        size: 500Gi
        class: gp3-csi
        deleteClaim: false
  resources:
    requests:
      cpu: "4"
      memory: 8Gi
    limits:
      cpu: "8"
      memory: 16Gi
  jvmOptions:
    -Xms: 6144m
    -Xmx: 6144m
----
