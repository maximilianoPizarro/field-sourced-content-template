== Bienvenido al Workshop

En este workshop explorarás cómo implementar un pipeline de *Change Data Capture (CDC)* completo sobre OpenShift usando las tecnologías del ecosistema de *Streams for Apache Kafka*.

=== Qué vas a aprender

* Desplegar un cluster *Kafka en modo KRaft* (sin Zookeeper) usando AMQ Streams
* Configurar *Debezium* para capturar cambios en PostgreSQL en tiempo real
* Usar *Apicurio Registry* como Schema Registry para validación de eventos
* Diseñar rutas de integración visualmente con *Kaoto* dentro de *DevSpaces*
* Monitorear el pipeline con *Grafana dashboards* y *Service Mesh (Kiali)*
* Gestionar Kafka desde *Developer Hub* con el plugin de Kafka
* Usar *Lightspeed (MCP)* para troubleshooting del pipeline

=== Componentes del Pipeline

[cols="1,3",options="header"]
|===
| Componente | Descripción

| image:kafka-icon.svg[Kafka,24] *Streams for Apache Kafka*
| Cluster Kafka en modo KRaft, topics, bridge HTTP y exporter de métricas

| image:debezium-icon.svg[Debezium,24] *Debezium*
| CDC connector que captura cambios de PostgreSQL en tiempo real

| image:apicurio-icon.svg[Apicurio,24] *Apicurio Registry*
| Schema Registry para validación de esquemas Avro/JSON/Protobuf

| image:camel-icon.svg[Camel,24] *Apache Camel + Kaoto*
| Rutas de integración diseñadas visualmente, consumiendo eventos CDC

| image:redhat-openshift-logo.svg[OpenShift,24] *Service Mesh + Grafana*
| Observabilidad end-to-end con dashboards y trazas en Kiali
|===
