== SonataFlow custom resource

The *Customer Onboarding* workflow is declared as a `SonataFlow` CR. The following illustrates the resource shape (events and start state); the deployed manifest in Git contains the full `states` graph.

[source,yaml]
----
apiVersion: sonataflow.org/v1alpha08
kind: SonataFlow
metadata:
  name: customer-onboarding
  namespace: kogito-bpm
  labels:
    app: customer-onboarding
    app.kubernetes.io/part-of: kogito-bpm
spec:
  flow:
    start: ReceiveCDCEvent
    events:
      - name: cdcCustomerEvent
        type: cdc.public.customers
        source: debezium/cdc
      - name: approvalCompleted
        type: approval.completed
        source: internal
    functions:
      - name: classifyCustomer
        type: expression
        operation: |
          {tier: (if .customer.amount >= 50000 then "premium" elif .customer.amount >= 10000 then "standard" else "basic" end)}
    states:
      - name: ReceiveCDCEvent
        type: event
        onEvents:
          - eventRefs:
              - cdcCustomerEvent
            eventDataFilter:
              data: ".payload.after"
              toStateData: ".customer"
        transition: ValidateCustomerData
      # ... additional states: validation, classification, approval, provisioning, metrics
----

== Workflow steps (high level)

. *ReceiveCDCEvent* — Consumes CloudEvents from `cdc.public.customers` (Debezium payload in the event body).
. *ValidateCustomerData* — Ensures required fields (for example email, name) are present.
. *ClassifyCustomer* — Derives tier (for example basic / standard / premium) from business data.
. *CheckApprovalRequired* — Routes premium flows through approval when configured.
. *RequestManagerApproval* — Publishes approval requests and waits for callback events when applicable.
. *ProvisionAccount* — Parallel work such as account creation and welcome notifications.
. *EmitOnboardingMetric* — Publishes completion metrics to `bpm.onboarding.metrics`.
