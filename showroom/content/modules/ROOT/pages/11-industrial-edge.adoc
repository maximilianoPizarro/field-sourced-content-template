The *Industrial Edge* stack is already deployed and running on this cluster. It simulates a full IoT manufacturing environment: sensors publish data via MQTT, events stream through Kafka, and an ML model detects anomalies — all managed by GitOps.

This section is hands-on: open the UIs, tail the logs, and trigger changes to see the platform react.

== What's running

[cols="1,2,1", options="header"]
|===
| Component | What it does | Access

| Machine Sensors (x4)
| Simulate vibration and temperature readings every 5 seconds via MQTT
| Pod logs (see below)

| AMQ Broker (x2)
| MQTT broker receiving sensor data at the factory edge
| Cluster-internal (`broker-amq-mqtt-all-0-svc`)

| Kafka Clusters (x3)
| Dev, Factory, and Data Lake clusters for event streaming and replication
| Cluster-internal (bootstrap services)

| Camel K
| MQTT→Kafka bridge and Kafka→S3 data lake integration
| Cluster-internal integrations

| Line Dashboard (dev)
| Real-time visualization of sensor data and anomaly alerts
| link:https://line-dashboard-industrial-edge-tst-all.{cluster_domain}[Dev Dashboard^]

| Line Dashboard (factory)
| Production factory dashboard
| link:https://line-dashboard-industrial-edge-stormshift-line-dashboard.{cluster_domain}[Factory Dashboard^]

| MinIO S3
| Object storage for the data lake (model training data, artifacts)
| link:https://minio-console-industrial-edge-ml-workspace.{cluster_domain}[MinIO Console^] (minioadmin / minioadmin)

| IoT Consumer (messaging)
| Backend that consumes MQTT sensor data and sends it to the ML inference service
| Cluster-internal (no web UI)
|===

== Quick look: are sensors streaming?

Run this to see live sensor output from the factory:

[source,bash]
----
oc logs -f deploy/machine-sensor-1 -n industrial-edge-stormshift-machine-sensor --tail=5
----

You should see JSON payloads with vibration readings every few seconds. Press `Ctrl+C` to stop.

== Architecture

All components run on this single cluster across separate namespaces:

[source,text]
----
┌─────────────────────────────────┐
│  industrial-edge-tst-all        │  Dev/Test: full IoT stack
│  (sensors, broker, kafka,       │
│   dashboard, anomaly detection) │
├─────────────────────────────────┤
│  industrial-edge-stormshift-*   │  Factory: sensors → MQTT → Kafka
│  (machine-sensor, messaging,    │
│   line-dashboard)               │
├─────────────────────────────────┤
│  industrial-edge-data-lake      │  Central: Kafka mirror + S3 store
├─────────────────────────────────┤
│  industrial-edge-ml-workspace   │  ML: MinIO + OpenShift AI
└─────────────────────────────────┘
----

Based on the https://validatedpatterns.io/patterns/industrial-edge/[Red Hat Validated Patterns^] Industrial Edge pattern, adapted for single-cluster deployment.
