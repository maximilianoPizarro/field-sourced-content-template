Sensor data flows from MQTT into Kafka via Camel K. In this step you will inspect the Kafka clusters, topics, and consume live messages.

image::edge-mfg-messaging-ml.png[Messaging and ML components,align=center]

== Kafka clusters in this deployment

There are 3 Kafka clusters serving different roles:

[source,bash]
----
oc get kafka -A --no-headers 2>/dev/null | grep industrial-edge
----

[cols="1,1,2", options="header"]
|===
| Cluster | Namespace | Role

| `dev-cluster`
| `industrial-edge-tst-all`
| Development/test environment

| `factory-cluster`
| `industrial-edge-stormshift-messaging`
| Factory edge — receives sensor data via MQTT→Kafka bridge

| `prod-cluster`
| `industrial-edge-data-lake`
| Central data lake — receives mirrored data from factory
|===

== List Kafka topics

[source,bash]
----
oc get kafkatopics -n industrial-edge-tst-all --no-headers
----

Key topics to look for:

* `iot-sensor-sw-vibration` — vibration sensor data
* `iot-sensor-sw-temperature` — temperature data (will have traffic after enabling)

== Consume live messages from a topic

Read the latest vibration events from the dev cluster:

[source,bash]
----
oc exec -n industrial-edge-tst-all dev-cluster-dev-0 -- \
  bin/kafka-console-consumer.sh \
  --bootstrap-server localhost:9092 \
  --topic iot-sensor-sw-vibration \
  --max-messages 5
----

You should see 5 sensor readings printed, then the consumer exits. Each message contains the raw vibration data from the machine sensors.

== Check the factory Kafka cluster

[source,bash]
----
oc get kafkatopics -n industrial-edge-stormshift-messaging --no-headers
----

Consume from the factory (note: factory topics are prefixed with the namespace):

[source,bash]
----
oc exec -n industrial-edge-stormshift-messaging factory-cluster-factory-0 -- \
  bin/kafka-console-consumer.sh \
  --bootstrap-server localhost:9092 \
  --topic industrial-edge-stormshift-messaging.iot-sensor-sw-vibration \
  --max-messages 3
----

== Inspect Kafka-to-S3 integration (Camel K)

The Camel K integration writes Kafka events to MinIO S3 for the data lake:

[source,bash]
----
oc get integration -n industrial-edge-tst-all
oc get integration -n industrial-edge-data-lake
----

If integrations show `Running`, data is flowing to S3. If `Building Kit`, the integration images are still being compiled (takes a few minutes on first deploy).

== Check KafkaMirrorMaker2 (data replication)

MirrorMaker2 replicates data from the factory Kafka to the central data lake:

[source,bash]
----
oc get kafkamirrormaker2 -n industrial-edge-stormshift-messaging
----
