= Step 12: Line Dashboard & Anomaly Detection

The Line Dashboard visualizes sensor data in real-time and displays ML anomaly alerts. Open it and watch the system work.

== Open the Dev Line Dashboard

[source,bash,subs=attributes+]
----
echo "https://line-dashboard-industrial-edge-tst-all.{cluster_domain}"
----

Or click: link:https://line-dashboard-industrial-edge-tst-all.{cluster_domain}[Dev Line Dashboard^]

Once the page loads:

. Click *Realtime Data* in the left navigation
. Wait 10–15 seconds for data to appear
. You should see *vibration* charts updating in real-time

NOTE: Temperature charts are empty because temperature sensors are disabled. You will enable them in xref:15-ie-mlops.adoc[step 13].

== Open the Factory Line Dashboard

The factory has its own dashboard instance:

link:https://line-dashboard-industrial-edge-stormshift-line-dashboard.{cluster_domain}[Factory Line Dashboard^]

This dashboard reads from the factory Kafka cluster and shows the same sensor types.

== Observe anomaly detection

The anomaly detection model runs inference on vibration data. Under normal operation:

* Every *2–3 minutes*, the model flags an anomaly
* An *alert* appears in the dashboard with the anomaly score
* This simulates real manufacturing scenarios where vibration patterns indicate potential equipment failure

Watch the dashboard for a couple of minutes and look for alert indicators.

== Inspect the anomaly detection service

[source,bash]
----
oc get inferenceservice -n industrial-edge-tst-all 2>/dev/null || echo "InferenceService not found in tst-all"
oc get pods -l app=anomaly-detection -n industrial-edge-tst-all 2>/dev/null
----

== Check the messaging UI

The messaging app provides a web UI to see MQTT messages. Open it:

link:https://messaging-industrial-edge-tst-all.{cluster_domain}[Messaging UI^]

This shows raw MQTT payloads as they arrive from the sensors.

== Compare dev vs factory

Both environments receive the same sensor data types but from different sensor pod instances:

[source,bash]
----
echo "=== Dev sensors ==="
oc get pods -l app=machine-sensor -n industrial-edge-tst-all --no-headers

echo "=== Factory sensors ==="
oc get pods -l app=machine-sensor -n industrial-edge-stormshift-machine-sensor --no-headers
----

== What you observed

* Real-time vibration data streaming on two dashboards (dev + factory)
* ML anomaly detection generating periodic alerts
* Raw MQTT messages visible in the messaging UI
* Identical architecture deployed in dev and factory namespaces
