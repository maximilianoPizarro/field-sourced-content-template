== Key architecture decisions (20K)

[cols="1,2", options="header"]
|===
| Decision | Rationale

| KRaft mode (no ZooKeeper)
| Reduces pod count, simplifies operations, removes ZK as SPOF

| 5 combined broker+controller nodes
| Sufficient for 8K msg/sec with RF=3 and 24 partitions

| 8-node MinIO with erasure coding
| EC:2 tolerates 2 failed nodes, 1.5Ti per node for data lake

| Hub + Edge topology
| Network segmentation, edge isolation, independent scaling

| MirrorMaker2 with 8 tasks/source
| Handles 13K msg/sec cross-cluster replication
|===

== What you learned

* Platform resource requirements scale from 67 pods / 41 vCPU (demo) to 118 pods / 157 vCPU (20K devices)
* CDC and Mirror consumers add significant read amplification on Kafka brokers (5x and 7x respectively)
* Production deployments require 2-4 OpenShift clusters depending on HA and DR requirements
* KRaft mode eliminates ZooKeeper, reducing complexity and pod count
* Infrastructure options are equivalent across AWS, Azure, and on-premise — only the provisioning model differs
