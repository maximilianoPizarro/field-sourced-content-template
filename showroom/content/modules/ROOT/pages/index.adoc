= Event-Driven Architecture Live
:page-layout: home

Welcome to *Event-Driven Architecture Live* — a hands-on, real-time Change Data Capture (CDC) pipeline running on Red Hat OpenShift.

This environment wires a production-style event backbone: relational data changes are captured at the database, streamed through Apache Kafka, processed by integration and workflow services, and observed end-to-end. Use the steps below to follow the data from insert to notification and beyond.

== What's running

[cols="1,2,1", options="header"]
|===
| Component | Role | Access

| PostgreSQL
| Source database with sample `customers` and `orders` data (simulated inserts on a schedule)
| Cluster-internal service (`cdc-postgresql` in `kafka-cdc`)

| Debezium
| PostgreSQL CDC connector reading the logical replication stream (WAL) and publishing change events
| Managed via Kafka Connect; inspect tasks in link:https://kafka-console-kafka-cdc.{cluster_domain}[Kafka Console^]

| Kafka (AMQ Streams / Strimzi)
| Durable log for CDC topics, DLQs, and Connect internal topics
| link:https://kafka-console-kafka-cdc.{cluster_domain}[Kafka Console^]

| Apache Camel
| Integration routes that transform and route events (e.g. notifications, side effects)
| Routes run in the Camel CDC workload; topics visible in the console

| Mailpit
| SMTP capture UI for HTTP sink notifications from the pipeline
| link:https://n8n-mailpit-openshift-lightspeed.{cluster_domain}[Mailpit UI^]

| SonataFlow (Kogito BPM)
| BPMN-driven workflow (e.g. customer onboarding) consuming pipeline events
| link:https://customer-onboarding-kogito-bpm.{cluster_domain}[Workflow API^]

| Apicurio Registry
| Schema registry for JSON/Avro-style artifacts and API metadata
| link:https://apicurio-registry-kafka-cdc.{cluster_domain}[Registry UI^]

| Grafana
| Dashboards and metrics for brokers, Connect, and pipeline health
| link:https://grafana-observability.{cluster_domain}[Grafana^]

| Red Hat Developer Hub
| Self-service portal, software templates, and golden paths
| link:https://backstage-developer-hub-developer-hub.{cluster_domain}[Developer Hub^]
|===

== Kafka cluster (high level)

The Strimzi `Kafka` custom resource defines the brokers, listeners, and replication settings used by this demo. The snippet below illustrates the shape of the cluster definition (version and storage may match your deployed release).

[source,yaml]
----
apiVersion: kafka.strimzi.io/v1beta2
kind: Kafka
metadata:
  name: cdc-cluster
  namespace: kafka-cdc
spec:
  kafka:
    version: 3.7.0
    replicas: 3
    listeners:
      - name: plain
        port: 9092
        type: internal
        tls: false
    storage:
      type: persistent-claim
      size: 10Gi
      deleteClaim: false
    config:
      offsets.topic.replication.factor: 3
      transaction.state.log.replication.factor: 3
      default.replication.factor: 3
      min.insync.replicas: 2
----
