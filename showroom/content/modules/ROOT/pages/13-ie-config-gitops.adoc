= Step 11: Explore Kafka Streams

Sensor data flows from MQTT into Kafka via Camel K. In this step you will inspect the Kafka clusters, topics, and consume live messages.

== List Kafka topics

[source,bash]
----

----

Key topics to look for:

* `iot-sensor-sw-vibration` â€” vibration sensor data
* `iot-sensor-sw-temperature` â€” temperature data (will have traffic after enabling)

== Consume live messages from a topic

Read the latest vibration events from the dev cluster:

[source,bash]
----
oc exec -n industrial-edge-tst-all dev-cluster-dev-0 -- \
  bin/kafka-console-consumer.sh \
  --bootstrap-server localhost:9092 \
  --topic iot-sensor-sw-vibration \
  --max-messages 5
----

You should see 5 sensor readings printed, then the consumer exits. Each message contains the raw vibration data from the machine sensors.

== Inspect Kafka-to-S3 integration (Camel K)

The Camel K integration writes Kafka events to MinIO S3 for the data lake:

[source,bash]
----
oc get integration -n industrial-edge-tst-all
oc get integration -n industrial-edge-data-lake
----

If integrations show `Running`, data is flowing to S3. If `Building Kit`, the integration images are still being compiled (takes a few minutes on first deploy).

== Check KafkaMirrorMaker2 (data replication)

MirrorMaker2 replicates data from the factory Kafka to the central data lake:

[source,bash]
----
oc get kafkamirrormaker2 -n industrial-edge-data-lake
----
