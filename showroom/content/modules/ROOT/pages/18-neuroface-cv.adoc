== Overview

The *NeuroFace CV Pipeline* integrates facial recognition with event-driven architecture. A webcam feed is processed by a FastAPI backend using *LBPH* (Local Binary Patterns Histograms) for face recognition, while *Intel OpenVINO Model Server* runs the `face-detection-retail-0005` model for detection. A Python-based *CV Processor* polls both services and publishes detection events to a Kafka topic, triggering email notifications through Mailpit.

== Architecture

[cols="1,2,1",options="header"]
|===
| Component | Role | Namespace

| *NeuroFace Frontend*
| Angular web UI for face registration and live detection via webcam
| `neuroface`

| *NeuroFace Backend*
| FastAPI + OpenCV LBPH recognizer — trains labels, exposes `/api/labels`
| `neuroface`

| *OpenVINO Model Server (OVMS)*
| Intel inference server serving `face-detection-retail-0005` (FP16) via REST/gRPC, deployed as KServe `InferenceService`
| `neuroface`

| *Model PVC*
| 1 Gi PersistentVolumeClaim storing `.xml/.bin` model files from Intel Model Zoo
| `neuroface`

| *Jupyter Workbench*
| OpenShift AI Notebook for model experimentation
| `neuroface`

| *CV Processor*
| Python (UBI9) — polls OVMS `/v1/config` and Backend `/api/labels` every 30 s, publishes JSON events to Kafka
| `kafka-cdc`

| *Kafka Topic*
| `cv.face.detections` on Strimzi `cdc-cluster` with SASL_SSL + SCRAM-SHA-512
| `kafka-cdc`

| *Kafka Consumer*
| Same Python process — consumes events and routes by `event_type`
| `kafka-cdc`

| *Mailpit*
| SMTP + HTTP API — receives formatted HTML email alerts
| `openshift-lightspeed`

| *YOLO PPE Serving*
| Ultralytics YOLOv8n — detects PPE items (hardhat, safety-vest, goggles) from webcam frames
| `neuroface`

| *PPE Kafka Topic*
| `cv.ppe.detections` — PPE compliance events published by the backend via `confluent-kafka`
| `kafka-cdc`
|===

== Event Flow

The CV Processor runs a continuous polling loop:

. *Poll OVMS* — `GET /v1/config` every 30 seconds to check model server status
. *Poll Backend* — `GET /api/labels` every 30 seconds to detect new registered faces
. *Produce to Kafka* — Publishes JSON events to `cv.face.detections` topic (SASL_SSL)
. *Consume from Kafka* — Same process reads events and routes by `event_type`
. *Send email* — Formats HTML and `POST /api/v1/send` to Mailpit

== Security

* *Kafka authentication*: SASL_SSL with SCRAM-SHA-512 (`cdc-user` credentials from Kubernetes secret)
* *Kafka Console OIDC*: Authenticated via Keycloak (RHBK) — `platformadmin` gets admin role, `userN` gets viewer role
* *OVMS*: Cluster-internal access only (no external route)
* *Inter-namespace traffic*: All Kafka connections use TLS

== OpenShift AI Integration

The NeuroFace model is served through *OpenShift AI* using KServe:

* *ServingRuntime*: `ovms-face-detection` — custom runtime based on OVMS image
* *InferenceService*: `face-detection-retail-0005` — RawDeployment mode, storage from PVC
* *Jupyter Workbench*: `neuroface-workbench` — for model experimentation and debugging

Access the OpenShift AI dashboard at link:https://rhods-dashboard-redhat-ods-applications.{cluster_domain}/projects/neuroface[OpenShift AI -> neuroface project^].

== PPE Safety Detection (YOLO)

The NeuroFace platform includes an optional *PPE Safety Detection* module that uses YOLOv8n to monitor whether operators are wearing required protective equipment. The webcam feed from the frontend is analyzed in real time, and each detection event is published to the Kafka topic `cv.ppe.detections` for downstream analysis in Jupyter notebooks.

For the full walkthrough — architecture, event schema, verification commands, and Granite LLM integration — see xref:20-ppe-safety.adoc[PPE Safety Detection].
